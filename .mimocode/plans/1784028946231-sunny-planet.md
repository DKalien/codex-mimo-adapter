# 全局重命名计划：opencode → mimo

## 范围

将项目中所有 `opencode` 引用重命名为 `mimo`，涉及约 **49 个文件**、**470+ 处修改**。

---

## 命名映射规则

| 旧名称 | 新名称 | 说明 |
|---|---|---|
| `codex-opencode-adapter` | `codex-mimo-adapter` | CLI 二进制名、文件名、token 前缀 |
| `codex_opencode_adapter` | `codex_mimo_adapter` | Rust crate 名、RUST_LOG 过滤器、SQLite 文件名 |
| `opencode_adapter` | `mimo_adapter` | 模型路由前缀 `mimo_adapter/<key>/<model>` |
| `opencode_go_adapter` | `mimo_adapter` | Codex provider 名称 |
| `opencode-go` | `mimo` | 模型前缀 `mimo/<model>` |
| `OPENCODE_GO_*` | `MIMO_API_*` | 环境变量 `MIMO_API_KEY`, `MIMO_API_BASE_URL` |
| `CODEX_OPENCODE_*` | `CODEX_MIMO_*` | 环境变量 `CODEX_MIMO_LOCAL_TOKEN` 等 |
| `.codex-opencode-adapter.env` | `.codex-mimo-adapter.env` | 项目环境文件 |
| `.codex-opencode-adapter/` | `.codex-mimo-adapter/` | 用户目录 |
| `.codex-opencode/` | `.codex-mimo/` | 状态数据库目录 |
| `OpenCodeGoClient` | `MimoClient` | Rust 结构体名 |

## 向后兼容考虑

**破坏性变更清单**：
1. CLI 二进制名变更 → 需要重新安装
2. 环境变量名变更 → 需要更新 `.env` 文件
3. 文件路径变更 → 旧项目需要重新 init
4. Provider 名称变更 → 需要更新 `~/.codex/config.toml`
5. 模型路由前缀变更 → 需要更新 agent TOML

**建议**：由于这是自托管项目，不需要向后兼容，直接全量重命名。

---

## 验证步骤

1. `cargo build --release` 编译通过
2. `cargo test` 所有测试通过
3. `codex-mimo-adapter init --api-key "<key>"` 初始化成功
4. `codex-mimo-adapter start` 启动成功
5. `codex-mimo-adapter check` 健康检查通过
6. 验证 `.codex-mimo-adapter.env` 中的环境变量名正确
7. 验证 `~/.codex/config.toml` 中的 provider 名称为 `mimo_adapter`

## 风险

- **高**：重命名后需要重新初始化所有项目
- **中**：测试文件数量多，需要仔细检查
- **低**：文档更新是纯文本替换
