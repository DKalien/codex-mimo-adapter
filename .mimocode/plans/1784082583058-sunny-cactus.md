# 项目开发模式分析报告

## 1. 开发模式概述

该项目有明确的开发模式，支持从源码直接运行、调试和测试。

## 2. 开发模式方式

### 2.1 从源码运行（推荐）
```powershell
# 直接使用cargo run
cargo run -- init --api-key "<your-mimo-token-plan-api-key>"
cargo run -- start

# 或者使用release模式
cargo run --release
```

### 2.2 开发脚本模式
```powershell
# 使用专门的开发脚本
./scripts/dev-run.ps1 -ApiKey "<your-mimo-token-plan-api-key>"

# 支持的参数：
# -ApiKey: MiMo Token Plan API密钥
# -BaseUrl: 上游API基础URL（默认：https://token-plan-cn.xiaomimimo.com/v1）
# -ListenHost: 监听主机（默认：127.0.0.1）
# -Port: 监听端口（默认：4010）
# -LocalToken: 本地访问令牌（默认：codex-mimo-local）
# -StateDb: 状态数据库路径（默认：.codex-mimo/state.sqlite）
# -MaxConcurrency: 最大并发数（默认：8）
# -Release: 使用release模式构建
```

### 2.3 调试模式
```powershell
# 启用详细日志
$env:RUST_LOG = "codex_mimo_adapter=debug"
cargo run --release
```

## 3. 开发工具链

### 3.1 构建系统
- 使用Rust和Cargo
- 配置在Cargo.toml中
- 支持debug和release两种构建模式

### 3.2 测试命令
```bash
# 运行单元测试
cargo test

# 运行真实上游smoke测试
MIMO_API_KEY="your-key" cargo test --test e2e_real_smoke test_e2e_real_validation_suite -- --ignored --nocapture
```

### 3.3 辅助脚本
| 脚本 | 用途 |
|------|------|
| `scripts/dev-run.ps1` | 开发环境启动脚本 |
| `scripts/run-real-smoke.ps1` | 真实上游smoke测试 |
| `scripts/check-local-adapter.ps1` | 本地适配器健康检查 |
| `scripts/clear-state.ps1` | 清理状态数据库 |

## 4. 开发环境配置

### 4.1 环境变量（开发模式下可配置）
| 变量 | 默认值 | 说明 |
|------|--------|------|
| `MIMO_API_KEY` | 必填 | MiMo Token Plan API密钥 |
| `CODEX_MIMO_HOST` | `127.0.0.1` | 监听主机 |
| `CODEX_MIMO_PORT` | `4010` | 监听端口 |
| `MIMO_API_BASE_URL` | `https://token-plan-cn.xiaomimimo.com/v1` | 上游API基础URL |
| `CODEX_MIMO_LOCAL_TOKEN` | 生成的本地令牌 | 本地访问令牌 |
| `RUST_LOG` | `codex_mimo_adapter=info` | 日志级别（可设为debug） |

### 4.2 配置文件
- `.codex-mimo-adapter.env`：项目级配置
- `~/.codex-mimo-adapter/project-registry.toml`：全局项目注册表

## 5. 开发流程建议

### 5.1 初次开发
1. 安装Rust工具链
2. 克隆项目
3. 设置环境变量
4. 使用`cargo run`或`dev-run.ps1`启动

### 5.2 日常开发
1. 修改代码
2. 使用`cargo run`快速测试
3. 使用`cargo run --release`进行性能测试
4. 运行`cargo test`验证功能

### 5.3 调试技巧
- 启用`RUST_LOG=codex_mimo_adapter=debug`查看详细日志
- 使用`codex-mimo-adapter check`验证适配器状态
- 检查SQLite状态数据库了解请求历史

## 6. 关键文件
- `Cargo.toml`：项目配置和依赖
- `scripts/dev-run.ps1`：开发启动脚本
- `docs/USAGE.zh-CN.md`：使用说明文档
- `docs/DIAGNOSTICS.md`：诊断和调试指南

## 7. 验证步骤
```powershell
# 1. 启动适配器
./scripts/dev-run.ps1 -ApiKey "<your-key>"

# 2. 检查健康状态
codex-mimo-adapter check
```

## 结论
该项目有完善的开发模式，支持快速原型开发、调试和测试。主要开发方式是通过`cargo run`或专门的`dev-run.ps1`脚本从源码运行。
