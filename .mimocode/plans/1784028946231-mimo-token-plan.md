# 适配器改造方案：OpenCode Go → MiMo Token Plan

## 目标

将 codex-opencode-adapter 从调用 OpenCode Go 网关改为直接调用小米 MiMo Token Plan API。

## 关键信息

| 项目 | 值 |
|---|---|
| MiMo API Base URL | `https://token-plan-cn.xiaomimimo.com/v1` |
| API 格式 | OpenAI 兼容（Chat Completions API） |
| 认证方式 | Bearer Token |
| 上游端点 | `POST /chat/completions`、`GET /models` |

## 改造范围

由于 MiMo 使用 OpenAI 兼容格式，**核心协议转换代码无需修改**。改造集中在配置层：

### 1. 修改默认 Base URL

**文件**: `src/config.rs`

```rust
// 修改前
const DEFAULT_UPSTREAM_BASE: &str = "https://opencode.ai/zen/go/v1";

// 修改后
const DEFAULT_UPSTREAM_BASE: &str = "https://token-plan-cn.xiaomimimo.com/v1";
```

### 2. 更新环境变量模板

**文件**: `.env.example`

```bash
# 修改前
OPENCODE_GO_BASE_URL=https://opencode.ai/zen/go/v1

# 修改后
OPENCODE_GO_BASE_URL=https://token-plan-cn.xiaomimimo.com/v1
```

### 3. 更新 Agent 模板中的模型名称

**文件**: `resources/templates/` 下的 4 个 `.toml` 文件

根据 MiMo Token Plan 支持的模型列表，更新 `model` 字段：

| 模板文件 | 当前模型 | 可选新模型 |
|---|---|---|
| `oss-flash.toml` | `opencode-go/deepseek-v4-flash` | `opencode-go/mimo-v2.5` 或保留 |
| `oss-mimo.toml` | `opencode-go/mimo-v2.5` | 保持不变 |
| `oss-minimax.toml` | `opencode-go/minimax-m3` | `opencode-go/mimo-v2.5-pro` 或保留 |
| `oss-pro.toml` | `opencode-go/deepseek-v4-pro` | `opencode-go/mimo-v2.5-pro` 或保留 |

### 4. 更新配置示例

**文件**: `config.toml.example`

更新 base_url 注释和默认值说明。

### 5. 更新文档

**文件**: `README.md`、`docs/USAGE.zh-CN.md`

- 更新 Quick Start 中的 API key 示例
- 更新 Base URL 说明
- 添加 MiMo Token Plan 配置说明

### 6. 更新错误处理（可选）

**文件**: `src/upstream.rs`

`extract_error_message` 函数已支持多种错误格式，无需修改。但可以添加 MiMo 特定错误格式的处理（如果存在）。

## 不需要修改的部分

- **协议转换逻辑** (`src/conversion/`)：Responses API ↔ Chat Completions API 转换保持不变
- **SSE 流处理**：MiMo 使用标准 OpenAI SSE 格式
- **工具调用格式**：标准 OpenAI tool_calls 格式
- **模型路由逻辑**：`opencode_adapter/<key>/<model>` 格式保持不变

## 实施步骤

1. 修改 `src/config.rs` 中的默认 Base URL
2. 更新 `.env.example` 模板
3. 根据 MiMo 支持的模型更新 agent 模板
4. 更新文档
5. 运行测试验证
6. 使用真实 API 进行 smoke 测试

## 验证方法

```powershell
# 1. 重新构建
cargo install --path . --force

# 2. 重新初始化项目
codex-opencode-adapter init --api-key "tp-c0vf71omi9quspkl4z6i10gvtniuvjlxb9vqeci4c6pphbeo"

# 3. 启动适配器
codex-opencode-adapter start

# 4. 验证健康状态
codex-opencode-adapter check

# 5. 真实 API 测试
curl -X POST http://127.0.0.1:4010/v1/responses \
  -H "Authorization: Bearer $(codex-opencode-adapter auth print-local-token)" \
  -H "Content-Type: application/json" \
  -d '{"model":"opencode_adapter/<project_key>/opencode-go/mimo-v2.5","input":"Hello"}'
```

## 风险评估

- **低风险**：MiMo 使用 OpenAI 兼容格式，现有转换逻辑可直接复用
- **需验证**：MiMo 的 reasoning/tool_call 格式是否与 OpenAI 完全一致
- **回退方案**：保留 OpenCode Go 配置选项，通过环境变量切换
